RUBRIQUE
    À propos du modèle objet Active Directory

DESCRIPTION COURTE
    Décrit le modèle objet du module Active Directory pour
    Windows PowerShell.

DESCRIPTION LONGUE
    Cette rubrique présente les classes du module Active Directory et
    leurs propriétés, qui sont utilisées pour modeler des attributs
    Active Directory réels. Elle décrit également la hiérarchie de
    classes construite à partir de son équivalent Active Directory.
    Le modèle objet établit une fondation de données pour toutes les
    opérations prise en charge par les applets de commande du module
    Active Directory.

  Hiérarchie de classes
    La liste suivante présente la hiérarchie de classes définie dans
    le modèle objet du module Active Directory avec l'héritage de
    classe indiqué par une mise en retrait. Ce modèle d'héritage
    permet aux applets de commande Active Directory d'accepter une
    plage de types d'objets comme entrée. Cela signifie, par
    exemple, que l'applet de commande Get-ADPrincipalGroupMembership
    peut accepter comme entrée tous les objets suivants : ADGroup,
    ADAccount, ADComputer, ADServiceAccount ou ADUser. Cela fonctionne
    en raison du modèle d'héritage et garantit qu'un objet ADUser
    a toutes les propriétés d'un objet ADPrincipal.

    Classes du module Active Directory
      La liste suivante présente chaque classe de module Active
      Directory à partir de la liste de la hiérarchie de classes.
      Chaque classe définit un ensemble de propriétés, parmi
      lesquelles des attributs LDAP récupérés par défaut et des
      propriétés créées spécifiquement pour le module Active
      Directory. Ces nouvelles propriétés sont dérivées d'un ou
      de plusieurs attributs LDAP comme décrit dans les listes de
      classes.

      Classe de niveau de base à partir de laquelle toutes les autres
      classes sont dérivées.
