РАЗДЕЛ
    Идентификация объектов Active Directory

КРАТКОЕ ОПИСАНИЕ
    Объекты модуля Active Directory для Windows PowerShell обладают 
    идентифицирующими их атрибутами, которые могут использоваться для 
    поиска и извлечения объектов. 

ПОЛНОЕ ОПИСАНИЕ
    Для идентификации объектов в Active Directory у каждого объекта есть 
    атрибуты, которые можно использовать в качестве идентификаторов. В 
    модуле Active Directory значение идентификатора объекта может 
    передаваться с помощью параметра Identity. Каждому типу объектов 
    соответствует свой собственный набор возможных типов и значений 
    параметра Identity. Дополнительные сведения об использовании параметра 
    Identity см. в подробном описании этого параметра заданного командлета.

    При поиске с помощью командлетов модуля Active Directory значения 
    параметров Identity, Server и Partition используются для 
    идентификации одного уникального объекта. Параметр Server 
    используется для указания сервера, к которому требуется подключиться. 
    Параметр Partition сужает поиск до указанного раздела. Параметр 
    Identity затем разрешается в один уникальный объект в разделе. 
    Если обнаруживается несколько объектов, соответствующих значению 
    параметра Identity, модуль Active Directory генерирует ошибку.

    Дополнительные сведения о параметрах Server и Partition см. в 
    разделах справки для тех командлетов, в которых они используются,
    например Get-ADUser, с помощью следующей команды.
    
    get-help Get-ADUser 

  Объекты и идентификаторы

    У каждого объекта есть список атрибутов, которые могут 
    использоваться в качестве идентификаторов этого объекта. Кроме 
    того, если объект унаследован от другого объекта, идентификаторы 
    родительского объекта также могут использоваться в качестве 
    идентификаторов дочернего объекта. Дополнительные сведения 
    об иерархии объектов Active Directory 
    см. в разделе about_Active_Directory_Object_Model.

    Примечание. При использовании командлетов поставщика Active 
    Directory в качестве идентификаторов могут использоваться только 
    различающееся имя или относительное различающееся имя объекта. Список 
    командлетов поставщика Active Directory см. в разделе ActiveDirectory.



    Ниже представлен список идентифицирующих атрибутов для различных типов 
    объектов.

    ADAccount
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя учетной записи SAM (sAMAccountName) 
      
    ADComputer
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя учетной записи SAM (sAMAccountName) 

    ADDirectoryServer
      Имя объекта сервера (name) 
        Для экземпляров AD LDS используется синтаксис имени 
          <имя_компьютера>$<имя_экземпляра>.
        Для других экземпляров Active Directory используйте значение 
          свойства имени

    ADDomain 
      Различающееся имя
      GUID
      Идентификатор безопасности
      DNS-имя домена 
      NetBIOS-имя домена 

    ADDomainController
      GUID (objectGUID)
      IPV4-адрес
      IPV6-адрес (глобальный) 
      DNS-имя узла (dNSHostName)
      Имя объекта сервера
      Различающееся имя объекта параметров NTDS объекта 
      различающегося имени сервера, представляющего контроллер домена
      GUID объекта параметров NTDS в разделе конфигурации

    ADFineGrainedPasswordPolicy
      Различающееся имя 
      GUID (objectGUID) 
      Имя (name) 
      
    ADForest
      Полное доменное имя
      GUID (objectGUID) 
      DNS-имя узла
      NetBIOS-имя

    ADGroup
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя учетной записи SAM (sAMAccountName) 

    ADObject
      Различающееся имя 
      GUID (objectGUID) 

    ADOptionalFeature
      Различающееся имя
      Имя (name)
      GUID компонента (featureGUID) 
      GUID (objectGUID) 

    ADOrganizationalUnit
      Различающееся имя 
      GUID (objectGUID) 

    ADPrincipal
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя учетной записи SAM (sAMAccountName) 

    ADServiceAccount
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя учетной записи SAM (sAMAccountName) 

    ADUser
      Различающееся имя 
      GUID (objectGUID) 
      Идентификатор безопасности (objectSid) 
      Имя пользователя SAM (sAMUserName) 


  Форматы идентификаторов

    Объекты модуля Active Directory могут идентифицироваться по различным 
    атрибутам. Ниже представлен список этих атрибутов, описаны их типы и 
    форматы.

    Различающееся имя 
      Пример: CN=InnaOzhogina,CN=Europe,CN=Users,DC=corp,DC=contoso,DC=com

    DNS-имя домена 
      Пример: redmond.corp.contoso.com

    DNS-имя узла (dNSHostName)
      Пример: corp-DC01.corp.contoso.com

    GUID компонента (featureGUID) 
      Пример: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    Полное доменное имя
      Пример: corp.contoso.com

    IPV6-адрес (глобальный) 
      Пример: 2001:4898:0:fff:200:5efe:157.59.132.61

    GUID (objectGUID) 
      Пример: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    IPV4-адрес
      Пример: 157.59.132.61

    NetBIOS-имя домена 
      Пример: redmond

    Имя объекта сервера
      Пример: corp-DC01$

    Имя учетной записи SAM (sAMAccountName) 
    	Пример: innaozhoginareports

    Идентификатор безопасности (objectSid) 
      Пример: S-1-5-21-3165297888-301567370-576410423-1103

    Имя
      Пример: Recycle Bin Feature
