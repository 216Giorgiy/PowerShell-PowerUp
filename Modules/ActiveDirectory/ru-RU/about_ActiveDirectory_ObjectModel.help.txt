РАЗДЕЛ
    Об объектной модели Active Directory

КРАТКОЕ ОПИСАНИЕ
    Описывает объектную модель модуля Active Directory для
    Windows PowerShell.

ПОЛНОЕ ОПИСАНИЕ
    В этом разделе описаны классы модуля Active Directory и их
    свойства, используемые для моделирования фактических атрибутов
    Active Directory. Кроме того, в этом разделе представлена
    иерархия классов, созданная в соответствии со структурой Active
    Directory. Объектная модель представляет собой основу для всех
    операций, поддерживаемых командлетами модуля Active Directory.

  Иерархия классов
    В следующем списке представлена иерархия классов, определенная в
    объектной модели модуля Active Directory. Наследование классов
    обозначается отступом. Данная модель наследования позволяет
    командлетам Active Directory принимать на вход различные типы
    объектов. Это, например, означает, что командлет 
    Get-ADPrincipalGroupMembership может принимать на вход любой 
    из следующих объектов: ADGroup, ADAccount, ADComputer, 
    ADServiceAccount и ADUser. Это возможно из-за модели наследования 
    и гарантирует, что у объекта ADUser есть все свойства объекта ADPrincipal.

    Классы модуля Active Directory
      Ниже показаны все классы модуля Active Directory из приведенной
      иерархии классов. В каждом из классов определяется набор
      свойств, некоторые из которых являются атрибутами LDAP,
      извлекаемыми по умолчанию, а некоторые - новыми свойствами,
      которые созданы специально для модуля Active Directory. Эти
      новые свойства наследуются от одного или нескольких атрибутов
      LDAP, как указано в списке классов.


    ADEntity
      Базовый класс, от которого наследуются все остальные классы.
