TEMA
    Modelo de objetos de Active Directory

DESCRIPCIÓN BREVE
    Describe el modelo de objetos del módulo de Active Directory para
    Windows PowerShell.

DESCRIPCIÓN DETALLADA
    Este tema explica las clases del módulo de Active Directory y las
    propiedades utilizadas para modelar atributos reales de Active
    Directory. Además, describe la jerarquía de clases construida
    a partir de su homóloga de Active Directory. El modelo de
    objetos establece un fundamento de datos para todas las
    operaciones admitidas por los cmdlets del módulo de
    Active Directory.

  Jerarquía de clases
    En la lista siguiente se muestra la jerarquía de clases definida
    en el modelo de objetos del módulo de Active Directory; la
    herencia de clases se indica aplicando sangría. Este modelo de
    herencia permite que los cmdlets de Active Directory acepten un
    intervalo de tipos de objeto como entrada. Esto significa, por
    ejemplo, que el cmdlet Get-ADPrincipalGroupMembership puede
    aceptar como entrada cualquiera de los objetos siguientes:
    ADGroup, ADAccount, ADComputer, ADServiceAccount o ADUser. El
    modelo de herencia permite que esto funcione, y se garantiza que
    un objeto ADUser tenga todas las propiedades de un
    objeto ADPrincipal.

      ADObject

    Clases del módulo de Active Directory
      En la lista siguiente se muestran todas las clases del módulo de
      Active Directory a partir de la lista de jerarquía de clases.
      Cada clase define un conjunto de propiedades; algunas son
      atributos de LDAP que se recuperan de forma predeterminada
      y otras son nuevas propiedades creadas específicamente para
      el módulo de Active Directory. Estas nuevas propiedades se
      derivan de uno o más atributos de LDAP como se describe en
      la lista de clases.

    ADEntity
      Clase de nivel base de la que se derivan todas las demás clases.
