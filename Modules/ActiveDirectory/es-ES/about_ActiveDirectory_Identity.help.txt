TEMA
    Identidad de objetos de Active Directory

DESCRIPCIÓN BREVE
    El módulo de Active Directory para los objetos de Windows 
    PowerShell tiene diversos atributos de identificación que se 
    utilizan para tareas de búsqueda y recuperación. 

DESCRIPCIÓN DETALLADA
     Para identificar los objetos en Active Directory, cada uno de 
     ellos tiene atributos que se pueden utilizar como identificadores.
     En el módulo de Active Directory, el valor de la identidad de un 
     objeto se puede pasar mediante el parámetro Identity. Cada tipo 
     de objeto tiene su propio conjunto de tipos y valores posibles para 
     ser utilizado por el parámetro Identity. Vea la descripción detallada 
     del parámetro Identity del cmdlet correspondiente para obtener más 
     información sobre su uso.

     Al realizar búsquedas con los cmdlets del módulo de Active Directory, se
     utiliza el valor del parámetro Identity, junto con los valores de los 
     parámetros Partition y Server, para identificar de manera singular un 
     objeto único. El parámetro Server se utiliza 
     para buscar con qué servidor hay que conectarse. El parámetro Partition 
     restringe la búsqueda aún más y la reduce a una partición concreta. 
     A continuación, el parámetro Identity se resuelve en un único objeto 
     de la partición. Si se busca más de un objeto mediante la resolución 
     de identidad, el módulo de Active Directory devuelve un error.

     Para obtener más información sobre los parámetros Server y 
     Partition, vea los temas de ayuda de los cmdlets individuales en 
     los que se utilizan, como Get-ADUser, escribiendo: get-help Get-ADUser 
 
     Objetos e identidades

     Cada objeto tiene una lista de atributos que se pueden utilizar como
     identidad de ese objeto. Además, si el objeto hereda de otro objeto, las
     identidades del objeto primario también se pueden utilizar como identidades
     del objeto secundario. Para obtener más información sobre la jerarquía de
     objetos de Active Directory, vea about_ActiveDirectory_ObjectModel.

     Nota: para los cmdlets de los proveedores de Active Directory, 
     únicamente se pueden usar como identidades el nombre distintivo 
     (DN) o el nombre distintivo relativo (RDN). Para obtener una lista de
     cmdlets de proveedores de Active Directory, vea ActiveDirectory.

  Formatos de identidades

    Los objetos del módulo de Active Directory poseen una serie de 
    atributos de identidad. Se muestran a continuación junto con sus 
    tipos y formatos.

    Nombre distintivo 
      Ejemplo: CN=EvaValverde,CN=Europe,CN=Users,DC=corp,DC=contoso,DC=com

    Nombre de dominio DNS 
      Ejemplo: redmond.corp.contoso.com

    Nombre de host DNS (dNSHostName)
      Ejemplo: corp-DC01.corp.contoso.com

    GUID de característica (featureGUID) 
      Ejemplo: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    Nombre de dominio completo
      Ejemplo: corp.contoso.com

    Dirección IPV6 global 
      Ejemplo: 2001:4898:0:fff:200:5efe:157.59.132.61

    GUID (objectGUID) 
      Ejemplo: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    Dirección IPV4
      Ejemplo:157.59.132.61

    Nombre de dominio NetBIOS 
      Ejemplo: redmond

    Nombre del objeto de servidor
      Ejemplo: corp-DC01$

    Nombre de cuenta SAM (Administrador de cuentas de seguridad) 
      (sAMAccountName) 
      Ejemplo: informesevavalverde

    Identificador de seguridad (objectSid) 
      Ejemplo: S-1-5-21-3165297888-301567370-576410423-1103

    Name
      Ejemplo: Característica de papelera de reciclaje
