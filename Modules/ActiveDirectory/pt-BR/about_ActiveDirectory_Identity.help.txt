TÓPICO
    Identidade de objeto do Active Directory

DESCRIÇÃO RESUMIDA
    O módulo do Active Directory para objetos do Windows PowerShell tem um 
    intervalo de atributos de identificação que são usados para pesquisa e 
    recuperação. 

DESCRIÇÃO LONGA
    Para identificar os objetos no Active Directory, cada objeto tem 
    atributos que podem ser usados como identificadores. No módulo do 
    Active Directory, o valor da identidade de um objeto pode ser 
    passado com o uso do parâmetro Identity. Cada tipo de objeto tem seu 
    próprio conjunto de possíveis tipos e valores para serem usados pelo parâmetro 
    Identity. Consulte a descrição detalhada do parâmetro Identity do cmdlet 
    específico para obter mais informações sobre seu uso.

    Ao pesquisar com os cmdlets de módulo do Active Directory, o valor do 
    parâmetro Identity, juntamente com os valores dos parâmetros Server e 
    Partition, é usado para identificar um único objeto de modo exclusivo.  
    O parâmetro Server é usado para localizar o servidor ao qual se conectar. 
    O parâmetro Partition refina mais a pesquisa para uma partição específica.  
    O parâmetro Identity é então resolvido como um único objeto exclusivo 
    na partição. 

    Se mais de um objeto for encontrado com o uso da resolução de 
    identidade, o módulo do Active Directory gerará um erro.

    Para obter mais informações sobre os parâmetros Server e Partition, 
    consulte os tópicos da Ajuda sobre os cmdlets individuais em que eles 
    são usados, como Get-ADUser, digitando:
    get-help Get-ADUser

  Objetos e identidades

    Cada objeto tem uma lista de atributos que podem ser usados como uma 
    identidade para aquele objeto. Além disso, se o objeto for herdado de outro
    objeto, as identidades do objeto pai também poderão ser usadas como as
    identidades do objeto filho. Para obter mais informações sobre hierarquia de 
    objetos do Active Directoy, consulte about_ActiveDirectory_ObjectModel.

    Observação: para cmdlets do Provedor do Active Directory, só o 'Nome 
    diferenciado' ou o 'Nome diferenciado relativo' de um objeto pode ser 
    usado como identidade. Para obter uma lista de cmdlets do Provedor do 
    Active Directory, consulte ActiveDirectory.

    Atributos de identidade

    A seguir é apresentada uma lista de atributos de identidade por 
    tipo de objeto.

    ADAccount
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome da Conta SAM (sAMAccountName) 
      
    ADComputer
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome da Conta SAM (gerente de contas de segurança) (sAMAccountName) 

    ADDirectoryServer
      Nome do objeto de servidor (name) 
        Para as instâncias do AD LDS, a sintaxe de um nome é 
          <nome-do-computador>$<nome-da-instância>
        Para outras instâncias do Active Directory, use o valor da 
          propriedade de nome.
      Nome Diferenciado do objeto de Configurações NTDS
      Nome Diferenciado do objeto do servidor que representa o 
        Servidor de diretório.
      GUID (objectGUID) do objeto de servidor na partição da configuração
      GUID (objectGUID) do objeto de configurações de NTDS na 
        partição da configuração

    ADDomain 
      Nome Diferenciado
      GUID
      Identificador de Segurança
      Nome de domínio DNS 
      Nome do domínio NetBIOS 

    ADDomainController
      GUID (objectGUID)
      IPV4Address
      IPV6Address global 
      Nome de Host DNS (dNSHostName)
      Nome do objeto de servidor
      DN (Nome Diferenciado) do objeto de Configurações NTDS do 
      servidor que representa o controlador de domínio
      GUID de objeto de configurações NTDS sob a partição de configuração
      GUID do objeto de servidor sob a partição de configuração Nome 
      Diferenciado do objeto de computador que representa o controlador de 
      domínio

    ADFineGrainedPasswordPolicy
      Nome Diferenciado 
      GUID (objectGUID) 
      Nome (name) 
      
    ADForest
      Nome de domínio totalmente qualificado
       
      Nome de host DNS
      Nome NetBIOS

    ADGroup
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome da Conta SAM (gerente de contas de segurança) (sAMAccountName) 

    ADObject
      Nome Diferenciado 
      GUID (objectGUID) 

    ADOptionalFeature
            Nome Diferenciado
      Nome (name)
      GUID de recurso (featureGUID) 
      GUID (objectGUID) 

    ADOrganizationalUnit
      Nome Diferenciado 
      GUID (objectGUID) 

    ADPrincipal
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome da Conta SAM (sAMAccountName) 

    ADServiceAccount
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome da Conta SAM (sAMAccountName) 

    ADUser
      Nome Diferenciado 
      GUID (objectGUID) 
      Identificador de Segurança (objectSid) 
      Nome do Usuário SAM (sAMUserName) 


  Formatos de identidades

    Os objetos do módulo do Active Directory têm vários atributos de 
    identidade. A seguir é apresentada uma lista deles, seus tipos e 
    formatos.

    Nome Diferenciado 
      Exemplo: CN=IsabelMartins,CN=Europe,CN=Users,DC=corp,DC=contoso,
      DC=com

    Nome de domínio DNS 
      Exemplo: redmond.corp.contoso.com

    Nome de Host DNS (dNSHostName)
      Exemplo: corp-DC01.corp.contoso.com

    GUID de recurso (featureGUID) 
      Exemplo: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    Nome de domínio totalmente qualificado
      Exemplo: corp.contoso.com

    IPV6Address global 
      Exemplo: 2001:4898:0:fff:200:5efe:157.59.132.61

    GUID (objectGUID) 
      Exemplo: 599c3d2e-f72d-4d20-8a88-030d99495f20 

    IPV4Address
      Exemplo: 157.59.132.61

    Nome do domínio NetBIOS 
      Exemplo: redmond

    Nome do objeto de servidor
      Exemplo: corp-DC01$

    Nome da Conta SAM (gerente de contas de segurança) (sAMAccountName)
      Exemplo: relatóriosisabelmartins

    Identificador de Segurança (objectSid) 
      Exemplo: S-1-5-21-3165297888-301567370-576410423-1103

    Name
      Exemplo: Recurso Lixeira
