TÓPICO
    Sobre o modelo de objeto do Active Directory

DESCRIÇÃO RESUMIDA
    Descreve o modelo de objeto do módulo do Active Directory para do
    Windows PowerShell.

DESCRIÇÃO LONGA
    Este tópico explica as classes do módulo do Active Directory e
    suas propriedades usadas para modelar atributos reais do Active
    Directory. Ele também descreve a hierarquia de classe construída
    com base em seu equivalente no Active Directory. O modelo de
    objeto estabelece uma base de dados para todas as operações com
    suporte dos cmdlets do módulo do Active Directory.

  Hierarquia de classe
    A lista a seguir mostra a hierarquia de classe definida no modelo
    de objeto do módulo do Active Directory com herança de classe
    implícita por recuo. Esse modelo de herança permite que os
    cmdlets do Active Directory aceitem uma gama de tipos de objeto
    como entrada. Isso significa, por exemplo, que o cmdlet 
    Get-ADPrincipalGroupMembership pode aceitar como entrada qualquer um 
    dos seguintes objetos: ADGroup, ADAccount, ADComputer, ADServiceAccount ou
    ADUser. Isso funciona devido ao modelo de herança e garante que
    um objeto ADUser tenha todas as propriedades de um objeto ADPrincipal.

    Classes do módulo do Active Directory
      A lista a seguir mostra cada classe do módulo do Active
      Directory da listagem de hierarquia de classe. Cada classe
      define um conjunto de propriedades, algumas das quais são
      atributos LDAP recuperados por padrão e outras são novas
      propriedades criadas especificamente para o módulo do Active
      Directory. Essas novas propriedades são derivadas de um ou
      mais atributos LDAP como descrito nas listagens de classe.

    ADEntity
      A classe de nível base da qual todas as outras classes
      são derivadas.
